6. Maintain and Scale

To handle varying loads, we'll use the Horizontal Pod Autoscaler (HPA) An API resource in Kubernetes that automatically updates a workload resource (such as a Deployment), with the aim of automatically scaling the workload to match demand. . The HPA automatically increases or decreases the number of pods based on metrics like CPU utilization.

HPA Manifest (hpa.yaml)

Add this file to your chart's `templates` directory. It configures the HPA to maintain an average CPU utilization of 50%, scaling between 1 and 10 pods.

apiVersion: autoscaling/v2 kind: HorizontalPodAutoscaler metadata: name: {{`{{ include "chat-app-chart.fullname" . }}`}}-hpa spec: scaleTargetRef: apiVersion: apps/v1 kind: Deployment name: {{`{{ include "chat-app-chart.fullname" . }}`}}-deployment minReplicas: 1 maxReplicas: 10 metrics: - type: Resource resource: name: cpu target: type: Utilization averageUtilization: 50
